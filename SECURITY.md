# Security Policy

## Supported Versions

Only the latest release is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

---

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue inside the platform, please do not disclose it publicly or open a GitHub issue. Instead, follow this responsible disclosure policy:

1. **Email Reports**: Send a detailed description of the vulnerability to the project maintainer.
2. **Details to Include**:
   - Description of the bug and its potential impact.
   - Detailed proof of concept (PoC) steps or script.
   - Suggested remediation steps if available.
3. **Response Time**: We will acknowledge receipt of your report within 48 hours and work on a patch promptly.
