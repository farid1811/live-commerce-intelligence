# Contributing to Live Commerce Intelligence

We welcome contributions from the open-source community! Whether you are fixing a bug, suggesting a feature, or correcting documentation, your help is highly appreciated.

---

## 1. Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it before making any contributions.

## 2. How to Contribute

### 2.1 Reporting Bugs
- Search existing issues to verify the bug has not been reported.
- Open a new issue with a clear title, description, steps to reproduce, and environment details (Python version, OS, browser).

### 2.2 Suggesting Enhancements
- Open a new issue describing the proposed enhancement, its benefits, and potential implementation strategies.

### 2.3 Pull Requests
1. **Fork the Repository**: Create a fork of this repository on GitHub.
2. **Clone Locally**: Clone your fork to your computer.
3. **Create Branch**: Create a descriptive branch for your edits:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Develop & Test**: Implement changes, verify formatting, and run the server locally to test pages.
5. **Commit Cleanly**: Follow standard git commit conventions:
   ```bash
   git commit -m "feat: implement real-time chart refresh controls"
   ```
6. **Push and PR**: Push to your branch and open a Pull Request (PR) against the `main` branch of this repository.

## 3. Coding Guidelines
- Follow PEP 8 style standards for Python code.
- Write modular, readable code following SOLID principles.
- Maintain comments and docstrings.
- Ensure that the thesis optimizer constraints ($\theta \ge 0$, bias $\ge 0$) inside `training_service.py` are strictly preserved.
